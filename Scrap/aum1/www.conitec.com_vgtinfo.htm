Support for Game Programming Starter Kit

Authoring System
for 3D Computer Games

With 3D GameStudio you can produce 3D realtime applications - demos, adventures, role playing, action, advertising, or racing games - without programming skills! The GameStudio development system contains the landscape editor WED with integrated ACKNEX 3D game engine and WDL script language support. You can create buildings, dungeons, landscapes, even whole cities, and populate them with your own creatures.
So be aware: It's no toy - it's suited not only for creating high quality games, but also for quickly presenting game concepts to potential publishers, for developing commercial applications, like virtual exibitions or warehouses, or even some 2-D applications - the engine is not restricted to 3-D! Look at the screenshots below, taken from actual ACKNEX games - read the FAQ file or follow the game links at the end of this page - or download one of the demos.

Game creating is so easy: At first describe the modes, user interface and basic objects for your game in a script file. Then build rooms, buildings, labyrinths, and landscapes on the screen, like with a CAD program. After that you can place objects, actors, and enemies. You have complete control over the behavior of anything in your level through simple script files. At last you can add intros, extros, panels, menus, cockpits, and displays.

System requirements: Required: 486DX-33, DOS 6.0 or above, standard VGA, 4 MB RAM. Recommended: Pentium, 16 MB RAM, Windows 95 with DirectX 5, or Windows 98. Game control with joystick, mouse and keyboard. Sound via Soundblaster, Adlib or Roland MPU.

Graphics: ACKNEX 3D engine, first or 3rd person 360° perspective. Buildings and landscapes with textured walls, floors and ceilings, stairs, backgrounds, things and actors. Light source at player position plus static light sources. Fog and underwater effects. Realtime 3-D engine renders backgrounds, floors, ceilings, walls and slopes at any height and any angle. Floors and ceilings may be tilted at any angle, and you can stack multiple floors one over each other to create bridges or floating objects! Regions and whole buildings may shift, lift, tilt or rotate during game play in real time.

Degrees of freedom: Forward/backward, left/right, up/down, rotation, tilt (up to nearly 90°), zoom. Walking, driving, flying, climbing, swimming, diving. User-definable motion dynamics with gravity, friction, inertia and acceleration.

Resolution: From 320x200 up to 800x600 Pixels, 256 colors.

Performance: 20 Frames/sec (486DX-66 with Local Bus VGA); 44 Frames/sec (Pentium-100 with PCI VGA); values for 320x200 picture size.

Textures: Animated bitmaps - PCX, LBM or FLI/FLC - for things, actors, walls, floors, or sky. Bitmap sizes up to 1024x1024 pixels. All textures can be scaled. Ambient lighting, shading, radiance, albedo for static light sources, transparent and translucent textures, shadow textures to create actor shadows, lightmaps to shape smooth shadows, attached textures for inscriptions or bullet holes.

Polygonal models:  Objects and actors can be built either from 2D sprites (up to 64 sides and animation cycles) or from textured, animated 3D polygons (MDL format).

Sound/Music: 8 channel polyphonic sound, stereo effect to localize objects and actors. MIDI player supports IBK format for FM instruments and drums, as well as MPU401 compatible wave table instruments.  CD audio is supported.

User interface: 2D overlays, 3D overlays (polygonal weapons), multiple 3D windows, text output, auto map, menus and user-defined panels with buttons, sliders, bar and digit displays. All objects can be manipulated by touching or clicking them with the mouse, e.g. to collect them in an inventory.

Intro / Titles: Full screen background pictures with animated parts and scrolling text; FLI or FLC animations with sound and music.

Additional Features: Demo save/replay mode; screenshot key; control of external devices (e.g. trackers) and I/O ports by script files; output of game values into a text file; 2-player mode via serial link, modem, network, or internet.

Objects: A game is constructed from objects. Objects of higher order (e.g. a wall) may contain such ones of lower order (e.g. a texture). The following objects are supported (from lower to higher order): Sound, song, instrument bank (IBK), bitmap, palette, FLIC, texture, attached texture, light map, polygon model, palette, font, string, text, message, overlay, mouse pointer, numeric display, bar graph display, picture display, button, slider, 2D slider, panel, automap, action, wall, thing, actor, genius (light source), way, region, map (topography), level. You can control each object's properties through up to 80 single parameters.

Scripts: The behavior of walls, regions, things, actors, weapons etc. as well as external devices (trackers, light shows) can be controlled by scripts. Scripts may be triggered through certain events (being hit or touched, coming near, being clicked on, being entered or left, time amount passed, key pressed etc.). Each actor is a state machine with script-controlled artificial intelligence.

Distribution: Games can be created for DOS as well as for Windows/DirectX. Integrated resource compiler packs all game files (sounds, bitmaps) together into a single compressed resource file. The end-user game runs directly from CD-ROM, the PC configuration (joystick, mouse, memory, sound hardware) is automatically detected, no installation is required.

Documentation: 100+ page English manual, 100+ page tutorial; Online user magazine with ads, hints, and game reports.
